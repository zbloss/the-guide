Background: We have now created a service called The Guide which starts by ingesting PDFs of Dungeons & Dragons (DnD) rules and campaign information. It then uses this knowledge base to provide question answering functionality to a dungeon master. The DM may ask anything like "What is the standard AC for a bugbear?" into a text chat, The Guide then answers the question using local LLMs (optionally cloud LLMs) and it's knowledge base. It returns the answer and references to the DM. We have built out backend APIs for this system in rust and now need to create a user interface to interact with this service. This frontend should be built in a web framework of your choosing. Understand that I plan to package the frontend and backend together into an application using [Tauri](https://v2.tauri.app/start/).

I have started by creating an empty tauri app in ./guide-frontend which has been configured to use React JS, Typescript, and Bun.

For now don't spend too much effort making the frontend look nice. It's much more important that the functionality provided by the backend guide rust services is exposed to the frontend.
